## Pre-mortem
如果 6 个月后 PM skill 失败：
- **死因1:** 14k+ Star 的 Task Master 成为事实标准，"为什么不用 Task Master？"无法回答
- **死因2:** Claude Code 内置 Tasks 持续迭代，覆盖 80% PM 需求，独立 skill 无存在必要
- **死因3:** 每个子 agent（PM/Architect/Reviewer）都有专门竞品做得更好，"一站式"体验不足以弥补单项劣势

## Cognitive Biases
- **Novelty**: "AI agent 生态这么热，PM skill 肯定有需求" — 实际上 25+ 竞品已经存在
- **Anchoring**: 从 preflight 的决策审查视角出发，锚定在"把 preflight 能力延伸到执行阶段"
- **Context blindness**: 忽略了 Claude Code Tasks 作为内置方案会吃掉大部分需求

## Opportunity Cost
开发独立 PM skill 会牺牲：preflight 自身能力迭代、新的差异化方向探索、与其他 skill 的集成工作。

## Recommendation
**🟡 PIVOT.** 不做独立的 PM skill（市场极度饱和），改为：
- **方向A:** 将 preflight 从"一次性检查点"升级为"持续决策哨兵"——在执行阶段定期触发检查（"方向还对吗？""有新竞品出现吗？""初始假设还成立吗？"）
- **方向B:** 聚焦 preflight 自身的缺陷修复和能力增强（本次审查发现的 8 个问题），做好 preflight 的核心竞争力
- 具体选择需要用户确认
